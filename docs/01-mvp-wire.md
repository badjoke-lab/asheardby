# AsHeardBy 画面構成ワイヤー案 v0.1

## 全体方針

MVPは **1ページ完結**。
ただし見た目は単なる縦長LPではなく、**百科事典の見開き1ページを読む感覚**に寄せる。

役割はこの順で固定する。

1. 何のサイトか分かる
2. 音源を選ぶ
3. フィルタを選ぶ
4. 聞き比べる
5. 目でも差を見る
6. 説明を読む
7. 限界を理解する

## デスクトップ基本構成

- Hero / Title band
- 左カラム: 音源選択 + フィルタ一覧
- 中央カラム: 再生プレイヤー + Compare toggle + 主視覚化 + 補助視覚化
- 右カラム: Notes / Explanation
- 下段: About / Notes strip

## モバイル構成

1. Hero
2. 音源選択
3. フィルタ選択
4. 再生プレイヤー
5. 主視覚化
6. 補助視覚化
7. フィルタ説明
8. About / Notes

## 初期体験

- Voice sample が初期選択
- High-frequency loss reference が初期選択
- Compare は Filtered 開始
- Original / Filtered を即時切替
- 初期状態は空にしない
