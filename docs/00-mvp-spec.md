# AsHeardBy 正式MVP仕様書 v0.1

## 1. プロジェクト定義

**AsHeardBy** は、**聞こえ方の違いをブラウザ上で比較・体験できるサイト**である。
現時点では **AsSeenBy と並ぶ独立リポジトリの単体プロジェクト**として進める。
将来的に TheirSense へ統合される前提は意識するが、**今は統合作業を主作業にしない**。

---

## 2. シリーズ全体の立ち位置

AsHeardBy を含む TheirSense 系列全体は、**厳密再現や診断を目的としない**。
基本思想は次のとおり。

> この件はこのように言われている・書かれている。
> それを、比較体験できる形にしてみる。

したがって AsHeardBy は、

* 医療的な診断ツールではない
* 個人の聞こえ方を正確に再現するものではない
* 一般に説明される特徴や傾向をもとにした
  **reference / approximation / translated experience**
  を提供する

という立場を取る。

---

## 3. AsHeardBy の立ち位置

AsHeardBy は **Condition + Reference の二層構造** とする。

### Condition

日常の聞き取り困難や状況差を、比較体験向けに表現する層。

### Reference

年齢差・性差・動物など、知覚差を参考比較として提示する層。

現時点では **Condition を中核** とするが、
初期段階から **Animal Reference も同居** させる。

---

## 4. MVPの目的

MVPでは、最低限次を成立させる。

1. サイト上で音声サンプルを再生できる
2. フィルタを切り替えて聞こえ方の違いを比較できる
3. 原音とフィルタ後を同一音源で比較できる
4. 何を表現しているか説明文がある
5. 変化を目でもわかる形で表示できる
6. UIが AsSeenBy 系の百科事典風世界観に沿っている
